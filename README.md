## 🛠 Функционал

### Дашборд (Vercel + Supabase)
- KPI: оборот, количество заказов, средний чек, крупные заказы (≥50k)
- График выручки по часам (Area Chart с градиентной заливкой)
- График выручки по городам (горизонтальный бар)
- Таблица заказов с раскрывающимся составом
- Блок «Крупные заказы» в стиле Telegram
- Автообновление каждые 30 секунд
- Адаптивная вёрстка, XSS-защита

### Telegram-бот (Python)
- Уведомления о новых заказах ≥ 50 000 ₸
- WebApp с дашбордом
- Команды: Полный отчёт, Статус, ТОП товаров, ТОП-5 сделок
- AI-аналитика через Groq (Llama 3.3 70B)
- Многопользовательский режим: пригласительные ссылки, управление доступом

### Синхронизация данных
- Скрипт `sync_crm.py` — получение заказов из RetailCRM API v5 и загрузка в Supabase
- Агрегация позиций в один заказ, сохранение города, менеджера, состава, `crm_id`
- Поддержка аргументов командной строки: `--config /путь/к/config.env`

## 🤖 Процесс разработки с AI-инструментами

Проект разрабатывался последовательно с использованием трёх AI-инструментов.

### 1. Gemini (бесплатная версия)
- **Задачи:** создание базовой структуры бота, подключение к Supabase.
- **Ограничения:** потеря контекста диалога через 3-4 сообщения.
- **Результат:** рабочий прототип, непригодный для дальнейшего масштабирования.

### 2. GitHub Copilot (бесплатный)
- **Задачи:** доработка функций, вёрстка дашборда.
- **Ограничения:** длина ответа не позволяет выдать целый файл, только фрагменты.
- **Результат:** дашборд заработал, но код был фрагментирован, дизайн минимальный.

### 3. Claude (Claude.ai)
- **Задачи:** полная переработка проекта, дизайн, безопасность, многопользовательский режим.
- **Ограничения:** лимит длины ответа (буфер обмена) — невозможно получить весь код одним сообщением.
- **Решение:** декомпозиция задачи на части, точечные правки функций, отладка через `console.log`.
- **Результат:** готовый продукт, соответствующий требованиям тестового задания.

**Ключевые промпты к Claude:**
> «Раздели код на 2 части для копирования, он не влезает в буфер обмена»
> «Добавь `console.log` в `loadData` для отладки загрузки данных из Supabase»
> «Сделай дизайн дашборда в стиле Nova Edition: тёмная тема, градиенты, шрифт Space Mono для цифр»
> «Добавь многопользовательский режим с пригласительными ссылками и возможностью удаления пользователей»
> «Исправь XSS-уязвимость — экранируй все пользовательские данные перед вставкой в HTML»

## 🧱 Технические проблемы и решения

| Проблема | Решение |
|----------|---------|
| Дашборд не видел данные после включения RLS в Supabase | Создана политика `CREATE POLICY "Enable read for all" ON orders FOR SELECT USING (true)` |
| Поле суммы называлось `total_summ` в старом коде и `total_sum` в новом | Написана универсальная функция `get_sum()` |
| Изменения в коде не отображались в браузере | Добавлен лоадер, автообновление, очистка кэша через `visibilitychange` |
| Toast-уведомления накладывались друг на друга | Установлен лимит `MAX_TOASTS = 3`, старые удаляются автоматически |
| Ссылка на заказ в CRM вела на 404 | В `sync_crm.py` добавлено сохранение внутреннего `crm_id` заказа |
| Состав заказа не отображался в Chrome (только в Acode) | Добавлено экранирование `escapeHtml()` и исправлена структура `items` |
